# DnD Build Forge ⚔️

Конструктор персонажей **D&D 5e (SRD 5.1)** с обратным подбором по навыкам и
шаблонным гайдом по отыгрышу. Полностью оффлайн, без бэкенда и без ИИ —
весь гайд собирается по правилам из тегов готового персонажа.

Тёмная фэнтези-эстетика (пергамент/тёмный камень, золото и бордовый),
адаптив под мобильные. Контент — **только SRD/CC-BY-4.0**, проект можно
свободно публиковать.

---

## Запуск

```bash
npm install        # ставит зависимости
npm run dev        # дев-сервер на http://localhost:5173
npm run build      # продакшн-сборка (tsc -b && vite build) в dist/
npm run preview    # предпросмотр собранного билда
npm run typecheck  # только проверка типов
```

> **Про реестр npm.** В проекте лежит `.npmrc` с `registry=https://registry.npmjs.org/`,
> потому что сборка велась в окружении с недоступным корпоративным reverse-proxy.
> Если у вас свой реестр — удалите `.npmrc` или замените строку.

## Стек

- **React 18 + Vite 5 + TypeScript** (строгий режим, `noUnusedLocals`/`noUnusedParameters`).
- **Tailwind CSS 3** — кастомная тёмно-фэнтезийная палитра и компоненты.
- **zustand** — состояние персонажа и навигация визарда.
- **localStorage** — персистентность между сессиями через безопасную обёртку
  с `try/catch` (приложение не падает, если хранилище недоступно).
- Шрифты Cinzel (заголовки) + EB Garamond (текст) с Google Fonts.

## Структура

```
src/
  types/         # все типы и теги (Ability, Role, Race, CharClass, Spell, Feat, CharacterState…)
  data/          # размеченный тегами SRD-контент
    skills.ts        # 18 навыков
    races.ts         # 6 рас
    classes.ts       # 6 классов + по одному подклассу, фичи 1–5 ур.
    backgrounds.ts   # 5 предысторий
    spells.ts        # заговоры + 1 круг для кастеров
    feats.ts         # подмножество черт SRD для окна 4 ур.
  lib/
    calculations.ts  # реактивные расчёты листа (КД, ОЗ, модификаторы, спасброски, навыки)
    leveling.ts      # ячейки заклинаний и дорожка уровней
    recommend.ts     # обратный фильтр «Помоги выбрать»
    guide.ts         # движок шаблонного гайда по тегам
    store.ts         # zustand-стор + шаги визарда
    storage.ts       # безопасная обёртка localStorage
    io.ts            # экспорт/импорт JSON
  components/
    steps/           # 10 шагов визарда
    ui/              # переиспользуемые карточки
    App / ProgressBar / LiveSheet / WizardStep / WizardNav / GuideView
```

## Что реализовано (по ТЗ)

**Флоу-визард с прогресс-баром** (навигация вперёд/назад, состояние пишется в
localStorage на каждом шаге, по полоске можно прыгать):

1. **Помоги выбрать** (опциональный обратный фильтр)
2. Раса → 3. Класс → 4. Характеристики → 5. Предыстория →
6. Навыки/владения → 7. Снаряжение → 8. Заклинания (только кастеры) →
9. Прокачка → 10. Итог.

**Обратный фильтр.** Пользователь задаёт желаемые навыки (мультивыбор), роль,
стиль (кастер/оружие/смешанный) и сложность. Приложение пересекает хотелки с
тегами связок **класс + раса + предыстория**, скорит и показывает топ-3 (по
одному на класс, чтобы не было трёх клонов). Для каждого навыка подсвечивается
**источник** («✓ через Плут (выбор)» / «предыстория Преступник» / «раса
Полуэльф»). Недостижимые/отсутствующие в топе хотелки дают мягкое
предупреждение с ближайшей альтернативой. Кнопка «Выбрать связку»
автоматически проставляет шаги визарда (раса/класс/подкласс/предыстория и
покрывающие навыки) и переводит на шаг характеристик.

**Характеристики** — три режима: **Point Buy** (27 очков, +/− с подсчётом
стоимости), **Standard Array** (15/14/13/12/10/8 по слотам), **4d6 drop
lowest** с рероллом и распределением. Итоги показываются с расовыми бонусами и
модификаторами. Для полуэльфа — выбор двух гибких +1.

**Живой лист персонажа** (всегда виден сбоку, на мобильных — сверху):
КД, ОЗ, инициатива, скорость, бонус мастерства, 6 характеристик с
модификаторами, спасброски, навыки с владением/экспертизой, СЛ и бонус атаки
заклинаний. Всё пересчитывается реактивно (чистые функции в `calculations.ts`).

**Прокачка 1→5.** Интерактивная дорожка: на каждом уровне — фичи (классовые +
подкласса), бонус мастерства, ячейки заклинаний, окна ASI/feat. Клик
раскрывает детали. Выбор подкласса показан на каноничном уровне. На 4 уровне —
выбор **ASI** (+2/+1+1 с распределением) или **черты** (7 черт SRD).

**Шаблонный гайд по отыгрышу** (финальный экран) — собирается условной логикой
из тегов готового персонажа (`role`, `isCaster`, класс, подкласс, теги
заклинаний), **без ИИ**. Четыре карточки: «В бою», «Вне боя», «Прокачка
наперёд», «Комбо». Гайд заметно различается — например, воин-танк (держать
линию, Всплеск действия), плут-универсал (Внезапная атака, Хитрое действие),
волшебник-контролёр (контроль первым ходом, концентрация).

**Итог и экспорт.** Красивый лист персонажа + блок гайда. Экспорт/импорт JSON
(с миграцией на дефолты, чтобы старые/частичные файлы не ломали приложение),
печать/«как карточка» (отдельные `@media print` стили — светлый пергамент,
скрытый интерфейс), кнопка «Создать нового».

## Принятые допущения / упрощения

Где SRD тонок — упрощали, не затягивая платный контент:

- **По одному подклассу на класс** (Чемпион, Школа Воплощения, Вор, Домен
  Жизни, Путь Берсерка, Коллегия Знания) — как ядро билда.
- **Расы** — берётся одна каноничная вариация подрасы из SRD (Высший эльф,
  Горный дварф, Ловкий полурослик) без полного дерева подрас.
- **Хиты** считаются по фиксированному среднему за уровень (стандартная опция
  SRD), а не по броску — для предсказуемого листа.
- **КД** выводится из выбранного набора снаряжения (по совпадению названий
  брони) + «Защита без доспехов» у варвара + «Доспехи мага» у волшебника, если
  заклинание известно.
- **Экспертиза плута** автоматически вешается на первые два выбранных навыка
  класса (порядок выбора определяет экспертизу).
- **Заклинания** — рабочее подмножество: заговоры + 1 круг для волшебника,
  клерика и барда. Бюджет «известных» упрощён до фиксированных чисел 1 уровня.
- **Бросок 4d6** использует обычный `Math.random()`; распределение бросков —
  ephemeral (после перезагрузки сохраняются итоговые значения, не сам пул).
- Гайд и фильтр держатся на ручной разметке тегов в `data/` — это by design.

## Что осталось «во второй заход»

- Заклинания шире: 2–3 круги, полные списки классов, расход ячеек на листе.
- Уровни 6–20 и полный список черт/особенностей.
- Мультикласс и расчёт мультиклассовых ячеек.
- Полное дерево подрас и больше подклассов на класс.
- Выбор боевого стиля воина/инструментов как влияющих на лист параметров.
- Ручная экспертиза/выбор владений вместо авто-правил.
